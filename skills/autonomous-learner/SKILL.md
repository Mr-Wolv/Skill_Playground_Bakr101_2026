# Autonomous Learner

Self-directed knowledge acquisition engine for exploring and mastering new domains.

## When to use

- You need to learn a new technology or concept
- You want to explore a domain systematically
- Preparing for a project in an unfamiliar area

## Instructions

1. **Define learning objectives** — what specifically do you need to understand? What do you want to be able to do?
2. **Gauge current level** — what do you already know? What are the gaps?
3. **Identify authoritative sources** — official docs, academic papers, recognized experts
4. **Create a learning path** — progressive topics from fundamentals to advanced
5. **For each topic**:
   - Read/watch the source material
   - Practice with examples
   - Test understanding (explain in your own words, apply to a different problem)
6. **Synthesize** — connect new knowledge with existing knowledge

## Learning path structure

```markdown
## Learning Path: [Topic]

### Level 1: Fundamentals
- [ ] [Resource] — what, why, basic concepts
- [ ] Mini-exercise: [short practical task]

### Level 2: Practical
- [ ] [Resource] — common patterns, best practices
- [ ] Exercise: [longer task applying the concepts]

### Level 3: Advanced
- [ ] [Resource] — edge cases, internals, tradeoffs
- [ ] Project: [realistic project that demonstrates mastery]
```

## Techniques

- **Feynman technique**: explain the concept in simple terms — if you can't, you don't understand it well enough
- **Spaced repetition**: review at increasing intervals to solidify long-term memory
- **Active recall**: test yourself instead of re-reading
- **Deliberate practice**: work on tasks at the edge of your ability
